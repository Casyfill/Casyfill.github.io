Developing the package using conda, sphynx, and pyscaffold.

Creating a module might be tedious, especially if we want to test and document it. Fortunately, there is a large set of tools and services that helps us to copy with such a task, installed properly. In fact, once package deployed correctly, everything else is very simple.

This memo is compiled partly from presentation and tutorial by Mohit Sharma, and several other sources, like this

I. Creating conda environment

  • go to the project folder
  • create env: conda create -n yourenvname python=x.x anaconda
  • activate env: source activate yourenvname
  • install packages:
    • sphinx: conda install -n yourenvname sphinx
    • pyscaffold: conda install -n yourenvname -c https://conda.anaconda.org/ioos pyscaffold
    • setuptools: conda install -n project setuptools

You can also add Tox as integration.

II. Create sceleton

  • lets generate module seceleton directory: putup -l 'mit' project --with-travis. Skeleton in this case starts with git initialized and test folder preapared for Travis integration.

III. Documentation

  • go to the folder
  • run sphinx-quickstart to generate html documentation
  • more info here

IV. Git Repository and Integration

  • now you can create repository on the gitHub and connect to local environment
  • to syncronise your repo with travis, comment line31, 32 in tests/travis_install.sh
  • setup coverall
  • enjoy (and start checking the documentation)
